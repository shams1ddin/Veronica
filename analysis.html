02 — Уязвимости и проблемы
🔐
Нет валидации API ключей при старте
Критично
Проблема: Если ключ в .env не задан — приложение молча стартует и падает только при первом запросе к AI. Непонятная ошибка вместо ясного сообщения.

Риск: Утечка ключей в git, отсутствие ключа не обнаруживается до первого пользователя.
# ❌ Сейчас — ключ может быть None, ошибка придёт позже GOOGLE_GEMINI_API_KEY = os.getenv("GOOGLE_GEMINI_API_KEY") genai.configure(api_key=GOOGLE_GEMINI_API_KEY) # None → тихая ошибка # ✅ Нужно — валидация при старте if not GOOGLE_GEMINI_API_KEY: raise ValueError("GOOGLE_GEMINI_API_KEY не задан в .env!")
🛡️
CORS разрешён для всех источников (allow_origins=["*"])
Критично
Проблема: Любой сайт в интернете может делать запросы к вашему API и тратить платные токены.
# ❌ Опасно app.add_middleware(CORSMiddleware, allow_origins=["*"], ...) # ✅ Безопасно ALLOWED = os.getenv("ALLOWED_ORIGINS", "http://localhost:8000").split(",") app.add_middleware(CORSMiddleware, allow_origins=ALLOWED, ...)

D1
⚡ Streaming ответов (SSE)
Приоритет 1 ~4–6 часов
Токены приходят в браузер по мере генерации, как в ChatGPT. Gemini SDK поддерживает streaming через generate_content(..., stream=True). Фронтенд получает куски через EventSource и дописывает текст в реальном времени.
1 Бэкенд — новый endpoint: добавить GET /veronica/stream в main.py. Вернуть StreamingResponse(generator(), media_type="text/event-stream"). Принимать query через query-параметры (GET, не POST — EventSource не умеет POST).
2 Gemini streaming: вызвать model.generate_content(prompt, stream=True), итерировать по chunk.text, yield каждый кусок как data: {json}\n\n. В конце yield data: [DONE]\n\n.
3 OpenRouter streaming: в _call_openrouter() добавить параметр stream=True в тело запроса. Читать aiohttp response построчно через async for line in resp.content. Парсить SSE-строки вида data: {...}.
4 Фронтенд — EventSource: в static/index.html заменить fetch('/veronica', {method:'POST'}) на new EventSource('/veronica/stream?query=...'). Создать пустой div для ответа до прихода первого токена.
5 Append-логика: в обработчике es.onmessage — парсить JSON, дописывать chunk.text в innerHTML. При получении [DONE] — закрыть EventSource, запустить MathJax.typesetPromise() на финальный элемент.
6 Убрать эффект печати: имеющийся typeText() симулирует стриминг через setTimeout. После внедрения настоящего SSE — удалить его, чтобы не было двойного эффекта. Оставить Markdown-парсинг в конце через marked.parse().
7 Обработка ошибок: добавить es.onerror — показывать сообщение об ошибке, закрывать соединение. Добавить таймаут: если за 30 сек нет данных — закрыть и показать fallback.
D2
🐳 Docker + Production деплой
Приоритет 2 ~1–2 дня
Упаковать проект в Docker-контейнер и выложить на сервер. После этого Veronica работает 24/7 в сети, а не только на вашем ноутбуке.
1 Dockerfile: создать в корне проекта. Базовый образ — python:3.12-slim. WORKDIR /app. COPY requirements.txt → pip install. COPY . . CMD uvicorn main:app --host 0.0.0.0 --port 8000. Проверить: docker build -t veronica .
2 .dockerignore: добавить .env, .git, __pycache__, *.pkl, veronica.db. БД и секреты не должны попасть в образ.
3 docker-compose.yml: сервис veronica с build: . и env_file: .env. Volumes: ./data:/app/data для персистентности БД. ports: 8000:8000. restart: unless-stopped.
4 Nginx reverse proxy: добавить сервис nginx в docker-compose. Конфиг: proxy_pass к veronica:8000. client_max_body_size 15m (больше MAX_UPLOAD_SIZE). gzip on для статики.
5 SSL через Certbot: добавить сервис certbot в docker-compose. Volumes: ./certbot/conf:/etc/letsencrypt. Запустить certbot certonly --webroot для вашего домена. Настроить cron на certbot renew каждые 60 дней.
6 Выбор хостинга: DigitalOcean Droplet ($6/мес, 1GB RAM) или Hetzner CX11 (€3.29/мес). Создать VPS, установить Docker + docker-compose, скопировать проект через git clone + scp .env.
7 GitHub Actions CI/CD: создать .github/workflows/deploy.yml. Триггер: push в main. Шаги: SSH на сервер → git pull → docker-compose up -d --build. Секреты (SSH ключ, IP сервера) добавить в Settings → Secrets.
8 Мониторинг: добавить /health endpoint (уже есть!) в UptimeRobot — бесплатный пинг каждые 5 минут с оповещением в Telegram при падении. Логи: docker-compose logs -f veronica.

🌍 PWA — Установка на телефон
Приоритет 3 ~3–4 часа
Progressive Web App превращает сайт в «нативное» приложение: иконка на рабочем столе, работа офлайн (из кэша), возможность push-уведомлений. Очень мало кода — огромный эффект на восприятие продукта.
1 manifest.json: создать static/manifest.json. Поля: name, short_name, start_url, display: "standalone", theme_color, background_color, icons (массив PNG 192×192 и 512×512 в static/icons/). Иконки можно сгенерировать через любой favicon-генератор.
2 Подключить manifest: в static/index.html добавить в <head>: <link rel="manifest" href="/static/manifest.json"> и <meta name="theme-color" content="#5759D9">.
3 Service Worker — файл: создать static/sw.js. В install-событии кэшировать: index.html, CSS, JS-библиотеки (marked, MathJax), иконки. Стратегия: Cache First для статики, Network First для API-запросов.
4 Регистрация SW: в конце index.html добавить if('serviceWorker' in navigator) navigator.serviceWorker.register('/static/sw.js'). Раздавать sw.js через FastAPI как статику.
5 Офлайн-страница: создать static/offline.html с сообщением «Нет подключения к интернету». В fetch-хендлере SW: при ошибке сети → вернуть offline.html вместо API-ответов.
6 Проверка: открыть Chrome DevTools → Lighthouse → запустить аудит PWA. Должно быть 100/100. Протестировать «Добавить на главный экран» на Android и iOS.

🎙️ Голосовой ввод и TTS
Приоритет 4 ~1 день
Пользователь говорит — Veronica слушает и отвечает голосом. Web Speech API работает прямо в браузере без бэкенда (для ввода). TTS через Gemini или браузерный speechSynthesis.
1 Кнопка микрофона: добавить кнопку 🎙 рядом с полем ввода в index.html. CSS: анимация пульсирования во время записи (красная точка + ripple-эффект). Атрибут title="Нажми и говори".
2 Web Speech API — ввод: в JS создать const recog = new SpeechRecognition(). Параметры: lang='ru-RU', continuous=false, interimResults=true. В onresult — вставлять текст в поле ввода в реальном времени. При onend — авто-отправка.
3 Fallback для Safari/Firefox: браузеры без SpeechRecognition → добавить кнопку «Записать аудио» (MediaRecorder API). Отправить blob на бэкенд POST /veronica/transcribe. На бэкенде — вызвать Gemini с audio-файлом (Gemini 2.5 Flash умеет аудио).
4 TTS — браузерный: добавить кнопку 🔊 рядом с каждым ответом. При клике: speechSynthesis.speak(new SpeechSynthesisUtterance(text)). Выбрать голос: voices.find(v => v.lang === 'ru-RU'). Стоп-кнопка: speechSynthesis.cancel().
5 TTS — через Gemini (качество выше): создать endpoint POST /veronica/tts в main.py. Вызвать Gemini с response_modalities=["audio"]. Вернуть base64 аудио. На фронтенде — воспроизвести через new Audio('data:audio/wav;base64,...').play().
6 Настройки голоса: добавить в UI выбор скорости (0.5x — 2x) и голоса. Сохранять предпочтения в localStorage. Автовоспроизведение ответа — опционально (toggle в настройках).

🧠 RAG — Умная память на документах
Приоритет 5 ~3–5 дней
Retrieval-Augmented Generation: документы разбиваются на части, векторизируются и хранятся в векторной БД. При вопросе — находятся самые релевантные части и передаются в контекст модели. Veronica «помнит» всё что ей загрузили.
1 Установка ChromaDB: добавить chromadb>=0.5.0 в requirements.txt. Инициализировать persistent client: chroma_client = chromadb.PersistentClient(path="./chroma_db"). Создать коллекцию documents при старте.
2 Chunking (нарезка): после извлечения текста из документа (уже есть в _extract_document_text) — разбить на chunks по 500 символов с overlap 100. Библиотека: langchain.text_splitter.RecursiveCharacterTextSplitter или написать вручную (10 строк).
3 Векторизация через Gemini Embeddings: вызвать genai.embed_content(model="models/text-embedding-004", content=chunk) для каждого chunk. Получить вектор размерности 768. Сохранить в ChromaDB: collection.add(documents=[chunk], embeddings=[vec], ids=[chunk_id], metadatas=[{"file_id": ...}]).
4 Новый endpoint upload: расширить /veronica/upload-document — после сохранения файла автоматически запускать индексацию через asyncio.create_task(index_document(path, file_id)). Показывать прогресс через отдельный GET /veronica/index-status/{file_id}.
5 Retrieval при каждом вопросе: в ask_veronica() — перед вызовом AI получить embedding вопроса, запросить ChromaDB: collection.query(query_embeddings=[q_vec], n_results=5). Собрать топ-5 chunks в строку context.
6 Инжекция контекста в промпт: собрать итоговый промпт: "Используй следующий контекст из документов:\n{context}\n\nВопрос: {query}". Если context пустой — использовать обычный промпт без RAG.
7 UI — показать источники: добавить в ответ Veronica блок «Источники» — список документов, из которых взяты chunks. На фронтенде отобразить как коллапсируемый блок под ответом.
8 Управление индексом: добавить endpoint DELETE /veronica/rag/{file_id} — удаляет все chunks документа из ChromaDB. Показать в UI кнопку «Удалить из памяти» рядом с каждым загруженным файлом.
D6
🖼️ Редактор изображений в чате
Приоритет 6 ~3–5 дней
После генерации изображения пользователь может его доработать: дорисовать элементы (inpainting), расширить кадр (outpainting), увеличить разрешение (upscaling). Уникальная функция — мало кто из чат-ботов это умеет.
1 Canvas-редактор на фронтенде: добавить modal с <canvas> элементом. Когда пользователь кликает на сгенерированное изображение — открывается редактор. Инструменты: кисть для маски (белый = область для inpaint), eraser, zoom.
2 Маска → бэкенд: при нажатии «Дорисовать» — экспортировать canvas маску как base64 PNG. Отправить на POST /veronica/inpaint вместе с оригинальным image_url и текстовым prompt.
3 Inpainting через Gemini: на бэкенде — отправить оригинал + маску в Gemini 2.0 Flash: model.generate_content([prompt, image_part, mask_part], generation_config={response_modalities:["image"]}). Gemini умеет редактировать изображения по маске.
4 Inpainting через FLUX.1-Fill: альтернатива — HuggingFace модель black-forest-labs/FLUX.1-Fill-dev. POST на https://api-inference.huggingface.co/models/... с полями image + mask_image + prompt.
5 Upscaling: добавить кнопку «Увеличить качество» под каждым изображением. Бэкенд endpoint POST /veronica/upscale. Использовать HuggingFace модель gokaygokay/Flux-4x-Upscaler или philz1337x/clarity-upscaler.
6 История версий: хранить все версии изображения в uploaded_files с суффиксами _v1, _v2 и т.д. В UI — показывать слайдер «До / После». Кнопка «Откатить к версии N».
7 Outpainting (расширение кадра): в canvas-редакторе добавить возможность увеличить размер холста (padding снаружи). Белые края автоматически становятся маской. Отправить расширенное изображение на тот же /inpaint endpoint с prompt «продолжи изображение».
Сводный порядок реализации
D1
Streaming SSE — 4–6 часов. Наибольший эффект на UX при минимальных усилиях. Пользователи сразу заметят разницу — ответы появляются мгновенно.
D2
Docker + Деплой — 1–2 дня. Без этого Veronica живёт только на вашем ноутбуке. После деплоя — реальные пользователи и обратная связь.
D3
PWA — 3–4 часа. Делать сразу после деплоя. Иконка на телефоне кардинально меняет восприятие продукта. Почти нулевые трудозатраты.
D4
Голосовой ввод + TTS — 1 день. На мобиле особенно критично. Браузерный Web Speech API — быстро, Gemini TTS — качественно.
D5
RAG на документах — 3–5 дней. Серьёзная функция — выделяет Veronica среди простых чат-ботов. Браться после того как есть пользователи.
D6
Редактор изображений — 3–5 дней. Сложно, но уникально. Мало кто из чат-ботов умеет редактировать свои же изображения.
